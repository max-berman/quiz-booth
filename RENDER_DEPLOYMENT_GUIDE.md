# Render.com Deployment Guide for Quiz Booth

This guide provides step-by-step instructions for deploying Quiz Booth to Render.com.

## Prerequisites

- Render.com account
- GitHub repository with your Quiz Booth code
- Firebase project with Firestore database
- DeepSeek API key

## Step 1: Prepare Your Repository

### 1.1 Merge the Render Deployment Branch

```bash
git checkout main
git merge feature/render-deployment
git push origin main
```

### 1.2 Verify Changes

- ✅ CORS middleware added to server
- ✅ API base URL configuration in query client
- ✅ Render Blueprint configuration (render.yaml)
- ✅ Environment variable template (.env.render.example)

## Step 2: Render.com Setup

### 2.1 Create Render Account

1. Go to [render.com](https://render.com)
2. Sign up with GitHub
3. Verify your email address

### 2.2 Connect GitHub Repository

1. In Render dashboard, click "New +" → "Blueprint"
2. Select your Quiz Booth repository
3. Render will automatically detect the render.yaml file

### 2.3 Configure Services

#### Backend Service (quiz-booth-backend)

- **Type**: Web Service
- **Runtime**: Node.js
- **Plan**: Starter ($7/month)
- **Region**: Choose closest to your users

#### Frontend Service (quiz-booth-frontend)

- **Type**: Static Site
- **Plan**: Starter (Free)
- **Build Command**: `cd client && npm install && npm run build`

#### Database (quiz-booth-db)

- **Type**: PostgreSQL
- **Plan**: Starter ($7/month)
- **Database Name**: quizbooth

## Step 3: Environment Variables Configuration

### 3.1 Backend Service Variables

Set these in the Render dashboard for the backend service:

```bash
NODE_ENV=production
PORT=10000
FRONTEND_URL=https://quiz-booth-frontend.onrender.com

# Firebase Admin
FIREBASE_PROJECT_ID=your_firebase_project_id
FIREBASE_PRIVATE_KEY=your_firebase_private_key
FIREBASE_CLIENT_EMAIL=your_firebase_client_email

# DeepSeek API
DEEPSEEK_API_KEY=your_deepseek_api_key

# Database (auto-generated by Render)
DATABASE_URL=postgresql://user:pass@host:port/db
```

### 3.2 Frontend Service Variables

Set these in the Render dashboard for the frontend service:

```bash
# Firebase Client Configuration
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_firebase_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id

# API Configuration
VITE_API_URL=https://quiz-booth-backend.onrender.com
```

## Step 4: Database Setup

### 4.1 Apply Database Migration

After the database is created, run the migration:

```bash
# Get the connection string from Render dashboard
psql $DATABASE_URL -f migrations/001_usage_tracking.sql
```

### 4.2 Verify Database Connection

The application includes automatic database connection testing. Check the logs to ensure it's working.

## Step 5: Deployment

### 5.1 Automatic Deployment

- Render will automatically deploy when you push to main
- Monitor the deployment logs in the Render dashboard
- Check for any build errors or environment variable issues

### 5.2 Manual Deployment (if needed)

```bash
# Trigger manual deployment from Render dashboard
# Or push a commit to trigger automatic deployment
git commit -m "Trigger Render deployment"
git push origin main
```

## Step 6: Testing

### 6.1 Health Check

Test the backend API:

```bash
curl https://quiz-booth-backend.onrender.com/api/health
```

### 6.2 Frontend Testing

1. Visit: `https://quiz-booth-frontend.onrender.com`
2. Test authentication flow
3. Test game creation
4. Test question generation
5. Verify usage tracking

### 6.3 CORS Testing

Ensure frontend can communicate with backend:

- Check browser console for CORS errors
- Verify API calls are successful

## Step 7: Custom Domain (Optional)

### 7.1 Configure Custom Domain

1. In Render dashboard, go to your static site
2. Click "Settings" → "Custom Domains"
3. Add your domain (e.g., `quizbooth.app`)
4. Update DNS records as instructed

### 7.2 SSL Certificate

- Render provides automatic SSL certificates
- No additional configuration needed

## Troubleshooting

### Common Issues

#### CORS Errors

- Verify `FRONTEND_URL` environment variable is set correctly
- Check that the frontend URL is included in the CORS origin list

#### Database Connection Issues

- Verify `DATABASE_URL` is set correctly
- Check that the migration script ran successfully
- Ensure the database is in the same region as the web service

#### Build Failures

- Check that all dependencies are in package.json
- Verify build commands work locally
- Check for TypeScript compilation errors

#### Environment Variables

- Ensure all required variables are set
- Check for typos in variable names
- Verify Firebase credentials are correct

### Logs and Monitoring

- Access logs via Render dashboard
- Set up alerting for errors
- Monitor performance metrics
